================================================================================
                    EVERY TIME YOU START A NEW FEATURE
================================================================================

STEP 1 — WRITE THE SPEC (30 minutes)
─────────────────────────────────────────────────────────────────────────────

In TERMINAL 2 (not Terminal 1 — dev server stays running):

cd /Users/sunil/Downloads/Eden.so

cp verify/spec/auth.md verify/spec/my-feature.md

# Open the file in your editor:
nano verify/spec/my-feature.md
# or open it in VS Code

# Edit these three sections:
# 1. ## Task — one sentence describing what you're building
# 2. ## Changed Files — list the files you'll modify
# 3. ## Acceptance Criteria — 4-5 observable behaviors (AC-1, AC-2, etc)

# Example of what an AC looks like:
# 
# ### AC-1: Button appears with correct text
# - The page shows a button that says "Save"
# - The button is blue and clickable (not disabled)
# 
# ### AC-2: Clicking button saves data
# - Clicking the button shows a success message
# - Refreshing the page still shows the saved data


STEP 2 — BUILD THE FEATURE (2-4 hours)
─────────────────────────────────────────────────────────────────────────────

In TERMINAL 2:

cd /Users/sunil/Downloads/Eden.so

claude

# In Claude Code, paste this prompt:
# "Build the feature described in verify/spec/my-feature.md
#  The AC file shows exactly what needs to happen."

# (Claude will ask for permission, chat with you, and build it)
# (review the code, make sure it looks right)
# (exit Claude when done — Cmd+C or 'exit')


STEP 3 — VERIFY IT WORKS (2 minutes)
─────────────────────────────────────────────────────────────────────────────

In TERMINAL 2:

cd /Users/sunil/Downloads/Eden.so

./verify/run.sh verify/spec/my-feature.md

# Wait for the verdict output. If you see:
#   ✅ All criteria passed
# Great! You're done for the day.

# If you see:
#   ❌ Some criteria failed
# Continue to Step 4.


STEP 4 — FIX ANY FAILURES (1-2 hours)
─────────────────────────────────────────────────────────────────────────────

Read the failure output. It tells you EXACTLY what failed.

Example failure:
  ❌ AC-2 FAILED
    Expected: "Save button is blue"
    Actual: "Save button is gray"
    Evidence: .verify/evidence/AC-2/screenshot-3.png

# Open the screenshot to see what happened.
# Fix your code in the editor (or ask Claude to fix it).

Re-run verification:

./verify/run.sh verify/spec/my-feature.md

# Repeat until all ACs are green ✅

================================================================================
                           COMMON MISTAKES
================================================================================

❌ I ran ./verify/run.sh but it said "dev server not reachable"
✅ Make sure npm run dev is running in TERMINAL 1. Check http://localhost:5173

❌ I wrote an AC that's too vague like "the button looks good"
✅ ACs must be specific: "the button text says 'Save' and it's blue"

❌ I built the feature without writing the spec first
✅ Go back: write the spec in verify/spec/my-feature.md, THEN build

❌ I'm getting errors I don't understand
✅ Look at the screenshot evidence: .verify/evidence/AC-1/screenshot-*.png
   They show exactly what the browser saw

❌ I want to verify multiple features at once
✅ Verify them one at a time: 
   ./verify/run.sh verify/spec/feature-1.md
   ./verify/run.sh verify/spec/feature-2.md
