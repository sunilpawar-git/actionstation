================================================================================
                  YOUR EXACT WORKFLOW (copy-paste these commands)

🎯 BEFORE YOU START
───────────────────────────────────────────────────────────────────────────────

Terminal 1 — KEEP RUNNING (don't touch this)
  $ cd /Users/sunil/Downloads/Eden.so
  $ npm run dev
  → You'll see: "VITE v5.X.X ready in XXX ms"
  → LEAVE THIS RUNNING

Terminal 2 — RUN YOUR COMMANDS HERE
  $ cd /Users/sunil/Downloads/Eden.so
  $ echo "I'll use this terminal for all commands below"


================================================================================
🔵 STEP 1: WRITE THE SPEC (30 minutes)
================================================================================

What you're doing:
  Writing down "what should happen" BEFORE you build anything

Copy-paste these commands in Terminal 2:

  cp verify/spec/auth.md verify/spec/MY_FEATURE_NAME.md

Example:
  cp verify/spec/auth.md verify/spec/add-dark-mode.md

Then:
  1. Open VS Code
  2. Find the new file: verify/spec/add-dark-mode.md
  3. Edit it:

     OLD TEXT                          NEW TEXT (your feature)
     ════════════════════════════════════════════════════════════════
     # Task                            # Task
     Google OAuth login                Add dark mode toggle

     ## Changed Files                  ## Changed Files
     - src/features/auth/...           - src/features/workspace/components/Header.tsx
     - src/features/auth/...           - src/styles/theme.css

     ## Acceptance Criteria            ## Acceptance Criteria

     ### AC-1: Login page renders      ### AC-1: Button appears
     - User visits /                   - Header shows "🌙 Dark" button
     - User sees login card            - Button is clickable

     ### AC-2: Sign-in works           ### AC-2: Toggle works
     - User clicks button              - Clicking button makes page dark
     - User is redirected              - Text becomes light
     ... etc

  4. SAVE the file (Cmd+S)

When done with Step 1:
  You have a file that describes EXACTLY what your feature should do.


================================================================================
🔵 STEP 2: BUILD THE FEATURE (2-4 hours)
================================================================================

What you're doing:
  Telling Claude Code to build it

Copy-paste these commands in Terminal 2:

  claude

This opens Claude Code interactive mode. You'll see a prompt. Type:

  Build the feature described in verify/spec/add-dark-mode.md
  Follow the spec exactly. Each AC must pass.

Claude will:
  • Ask for permission to edit files
  • Write the code
  • Save files
  • Show you what changed

When Claude is done (or when you're happy with the changes):
  Press Ctrl+C
  
That's it. Step 2 complete.


================================================================================
🔵 STEP 3: VERIFY IT WORKS (2 minutes)
================================================================================

What you're doing:
  Automated test that checks if your feature matches the spec

Copy-paste this command in Terminal 2:

  ./verify/run.sh verify/spec/add-dark-mode.md

This runs:
  1. Pre-flight checks (bash)          [fast]
  2. Planning phase (Opus AI)          [takes screenshots in browser]
  3. Testing phase (Sonnet AI × 4)     [automated browser testing]
  4. Judge phase (Opus AI)             [reads evidence, makes verdict]

Wait for the output. You'll see:

OPTION A - Everything passed ✅

  ═══════════════════════════════════════════════════════════════
  AC-1  ✅ PASSED  Dark mode button appears in header
  AC-2  ✅ PASSED  Clicking button enables dark mode
  AC-3  ✅ PASSED  Dark mode persists after refresh
  ═══════════════════════════════════════════════════════════════
  ✅  All criteria passed. Safe to merge.

→ YOU'RE DONE! Go celebrate.


OPTION B - Something failed ❌

  ═══════════════════════════════════════════════════════════════
  AC-1  ✅ PASSED  Dark mode button appears
  AC-2  ❌ FAILED  Background still white after click
       Evidence: .verify/evidence/AC-2/screenshot-2.png
  AC-3  ⚠️  NEEDS-REVIEW  Persist check unclear
  ═══════════════════════════════════════════════════════════════

→ Go to STEP 4


================================================================================
🔵 STEP 4: FIX FAILURES (repeat until green)
================================================================================

If you saw ❌ or ⚠️ above, do this:

1. LOOK AT THE SCREENSHOT

   cat .verify/evidence/AC-2/screenshot-2.png

   (This shows what the browser actually saw)

2. FIX THE CODE

   Open the file in VS Code
   Fix whatever was wrong

3. RE-RUN VERIFICATION

   ./verify/run.sh verify/spec/add-dark-mode.md

4. CHECK THE NEW VERDICT

   If green ✅: Go celebrate
   If still red ❌: Fix again and re-run

Repeat steps 1-4 until everything is ✅


================================================================================
⚡ THAT'S IT
================================================================================

You just completed the entire workflow:

  Step 1: Write spec (what should happen)
  Step 2: Build feature (Claude builds it)
  Step 3: Verify (automated test)
  Step 4: Fix (if needed)

Every time you work on a feature, repeat these 4 steps.

The key insight:
  Spec FIRST (step 1)  ← This is the hard part
  Build SECOND (step 2)
  Verify THIRD (step 3)
  Fix FOURTH (step 4)

NOT: "build first, spec never"
